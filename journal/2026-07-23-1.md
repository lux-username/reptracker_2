## What was done

- Brainstormed alternatives to "Representative Tracker", the concern being that "tracker"
  frames the representative as a subject under watch, inverting the product's actual stance.
- **Owner caught a factual error in my reasoning:** I justified a "Brief"-family name by
  claiming "brief implies digest, which is literally what the LLM produces." There is no
  LLM — retired in session 6 (`lib/summaries.ts:1` says so in a comment; no `anthropic`
  dependency; `spec.md:120` marks the per-rep digest Retired). The name was defended from a
  stale line in `CLAUDE.md` rather than from the code.
- Traced the error to its source: `CLAUDE.md:12` still described the Anthropic/Haiku stack.
  Fixed it, then ran a subagent audit of the rest of `CLAUDE.md` against code + `STATUS.md` +
  `spec.md`. Five more findings, three substantive:
  - the scraping/cron bullet was wrong on both halves (cheerio is in `lib/`, not a route
    handler; the real scheduler is GitHub Actions, not Vercel Cron)
  - "one page, one route" is now two pages + two route handlers
  - the project description still used the "upcoming decisions" phrasing retired by #30
- Rewrote `CLAUDE.md`: **Stack section replaced with a pointer to `spec.md`** (see decisions),
  project description reworded off "upcoming decisions" with a note explaining why, three rows
  added to the Derived Facts table (latest commit / open issues / active milestone), and
  `sessions/` added to the one-home routing rule.
- Researched name components properly instead of from intuition — congressional publication
  vocabulary, parliamentary terms, etymologies, and the existing civic-tech field. Findings
  that changed the answer: "activity" is the House's own noun (house.gov/legislative-activity);
  "docket" historically meant *summary* and currently means *awaiting decision*, so it fails
  both constraints at once; "track" is also the crowded word (GovTrack since 2004).
- Filed the rename as a decision issue rather than implementing it, at owner's instruction.
- Confirmed `BrandMark` is pure geometry with no wordmark, so a rename doesn't touch it —
  and noticed in passing that its SVG is duplicated in `app/icon.svg`.

## Decisions taken (owner input)
- **Detection source:** authoritative calendar + corroborate; a bare data-gap heuristic
  never decides alone.
- **Granularity:** per-chamber (House vs. Senate detected independently — a rep's card
  and their senators' cards can differ).
- **Scope:** #8 + #27 together.
- **Copy:** minimal / factual only — no unverifiable "is home in their district" claim
  (wrong for senators anyway), no manufactured "call about now" urgency. Neutral bill
  heading, just repositioned. (Full rationale → decisions.md.)
- **Recess lead content:** contact + existing sponsored bills now; #21 (bills waiting in
  committee) becomes the true recess lead once built — committee assignments are stable
  across a recess.

  calendar **only as PDF/JPG** (no XML/table), so no machine-readable House return date →
  the House "until [date]" is omitted; filed #29 for the optional PDF path.
- Built `lib/session-status.ts` (pure parsers + cached I/O, mirrors floor-schedule.ts):
  per-chamber `{ inSession, returnDate }`. Senate from the schedule XML + floor
  next-convene; House from the weekly floor XML `weekOf` already scraped for #4.
- Wired `page.tsx → AddressLookup → RepSection` (per rep's chamber) + `FloorThisWeek`
  recess states + `prewarm.ts` Senate-calendar warm. Spec §"Recess behavior" rewritten
  to match.
